# Launch Checklist — Распечатка

Список задач перед запуском и сразу после деплоя.
Отмечайте пункты по мере выполнения.

---

## До деплоя

### Контакты и реквизиты

- [ ] Заполнить **ФИО владельца** в юридических страницах:
  - `src/app/privacy/page.tsx` — `[ЗАПОЛНИТЬ: ФИО физического лица — владельца сайта]`
  - `src/app/terms/page.tsx` — `[ЗАПОЛНИТЬ: ФИО владельца / реквизиты ИП или ООО]`
  - `src/app/consent/page.tsx` — `[ЗАПОЛНИТЬ: ФИО владельца сайта]`
  - `src/app/personal-data-consent/page.tsx` — `[ЗАПОЛНИТЬ: ФИО владельца сайта]`
  - `src/app/contacts/page.tsx` — `[ЗАПОЛНИТЬ: ФИО владельца]`
- [ ] Заполнить хостинг-провайдера в `src/app/privacy/page.tsx` — `[ЗАПОЛНИТЬ: указать конкретного хостинг-провайдера]`
- [ ] Заполнить реквизиты для оплаты в `src/app/terms/page.tsx` — `[ЗАПОЛНИТЬ: реквизиты для оплаты]` (после регистрации ИП/ООО)
- [ ] Заполнить стоимость доставки в `src/app/delivery-and-return/page.tsx`
- [ ] Проверить/обновить телефон и email в `src/data/site.ts`
- [ ] Заполнить или убрать пустые ссылки `social.instagram` и `social.vk` в `src/data/site.ts`
- [ ] Заполнить рейтинг и число отзывов Avito в `src/data/site.ts` → `platforms.avito`

### Юридические страницы

- [ ] Проверить юридические страницы с юристом (особенно для физлица без ИП/ООО)
- [ ] Убедиться, что все `[ЗАПОЛНИТЬ]` заполнены перед публикацией
- [ ] Проверить, что `/privacy`, `/consent`, `/terms` открываются корректно

### Production ENV

- [ ] Создать `.env.local` (или переменные в хостинге) на основе `.env.example`
- [ ] Заполнить `NEXT_PUBLIC_SITE_URL=https://ваш-домен.ru` (реальный домен, https)
- [ ] Сгенерировать `ADMIN_SESSION_SECRET` (минимум 32 символа, рекомендуется 64):
  ```bash
  node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
  ```
- [ ] Заполнить `ADMIN_PASSWORD` (минимум 12 символов, не угадываемый)
- [ ] Заполнить `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` для доставки заявок
- [ ] Указать `PRINTLAB_DATA_DIR` — путь вне директории проекта (например `/var/data/raspechatka`)
- [ ] Убедиться, что `NEXT_PUBLIC_YANDEX_METRIKA_ID` заполнен (если Метрика нужна сейчас)

### Безопасность перед деплоем

- [ ] Убедиться, что `.env`, `.env.local` НЕ попали в git (`git status`)
- [ ] Убедиться, что `data/orders.json` НЕ попал в git
- [ ] Убедиться, что папка `data/order-files/` НЕ попала в git
- [ ] Убедиться, что все `*.bak` файлы в `.gitignore`
- [ ] Проверить `git log --all -- .env` — убедиться, что секреты никогда не коммитились

### Домен и хостинг

- [ ] DNS настроен, домен указывает на хостинг
- [ ] SSL-сертификат активен (https работает)
- [ ] Выбрать хостинг с серверами в РФ (требование ФЗ-152 для хранения ПД граждан РФ)
- [ ] Убедиться, что `PRINTLAB_DATA_DIR` существует и доступен для записи на сервере

## После деплоя

### Базовая проверка

- [ ] Открыть главную страницу — убедиться, что загружается без ошибок
- [ ] Проверить мобильную версию (≤375px и ≤430px)
- [ ] Проверить продуктовую страницу (`/product/[slug]`)
- [ ] Проверить каталог (`/catalog`)
- [ ] Проверить конструктор (`/configurator`)
- [ ] Открыть `/privacy`, `/consent`, `/terms` — убедиться, что страницы доступны и нет `[ЗАПОЛНИТЬ]`

### Форма заказа

- [ ] Войти в `/admin` (логин/пароль из ENV)
- [ ] Отправить одну **контрольную** заявку через форму продуктовой страницы
- [ ] Проверить, что заявка появилась в `/admin → Заявки`
- [ ] Проверить загрузку изображения для фронта и бека (конфигуратор)
- [ ] Проверить превью обеих сторон в конфигураторе
- [ ] Проверить, что ZIP-архив с файлами формируется и скачивается (если реализовано)

### Telegram

- [ ] Убедиться, что Telegram-уведомление о контрольной заявке пришло в нужный чат
- [ ] Проверить форматирование сообщения
- [ ] Удалить или пометить контрольную заявку в `/admin` как тестовую

### SEO и индексация

- [ ] Открыть `/sitemap.xml` — убедиться, что URL содержат production-домен (не `example.com`, не `localhost`)
- [ ] Открыть `/robots.txt` — убедиться, что `Sitemap:` указывает на production-URL
- [ ] Открыть страницу в браузере → View Source → проверить `<link rel="canonical">` — должен быть production-домен
- [ ] Проверить Open Graph теги (`og:url`) через https://vk.com/dev/pages_parser или аналог
- [ ] Добавить сайт в **Яндекс Вебмастер**: https://webmaster.yandex.ru
- [ ] Подтвердить права на сайт (мета-тег или файл)
- [ ] Отправить sitemap в Яндекс Вебмастер

### Яндекс Метрика

- [ ] Убедиться, что счётчик создан на https://metrika.yandex.ru
- [ ] Заполнить `NEXT_PUBLIC_YANDEX_METRIKA_ID` в ENV и передеплоить (если не сделано ранее)
- [ ] Открыть сайт — в Метрике должно появиться посещение в режиме реального времени
- [ ] Принять cookie-баннер на сайте — проверить, что `analytics` согласие активно
- [ ] Проверить, что цели (order_submit_success и другие) появляются при тестовом оформлении заявки

### Бизнес-присутствие

- [ ] Создать или обновить карточку **Яндекс Бизнеса**: https://business.yandex.ru
- [ ] Убедиться, что контакты в карточке совпадают с сайтом
- [ ] Начать собирать реальные отзывы (просить клиентов оставлять на Яндекс.Картах)
- [ ] НЕ добавлять выдуманные отзывы на сайт

---

## Запрещено

- Коммитить `.env` или `.env.local` в git
- Коммитить `data/orders.json` (реальные заявки клиентов)
- Коммитить загруженные файлы клиентов (`data/order-files/`)
- Публиковать токены, пароли и секреты в открытом виде
- Использовать выдуманные или накрученные отзывы
- Оставлять `[ЗАПОЛНИТЬ]` placeholders на опубликованном сайте

## Что нужно запросить у владельца для финального заполнения

| Данные | Где используется |
|--------|-----------------|
| ФИО физического лица (или реквизиты ИП/ООО после регистрации) | `/privacy`, `/terms`, `/consent`, `/contacts` |
| Название хостинг-провайдера и подтверждение серверов в РФ | `/privacy` раздел 7 |
| Реквизиты для оплаты (после регистрации ИП/ООО) | `/terms` раздел 3 |
| Стоимость доставки по Москве и регионам | `/delivery-and-return` |
| Рейтинг и число отзывов на Avito (если нужен бейдж) | `src/data/site.ts` → `platforms.avito` |
| Ссылки на Instagram и VK (если есть страницы) | `src/data/site.ts` → `social.instagram`, `social.vk` |
| Номер счётчика Яндекс Метрики | `NEXT_PUBLIC_YANDEX_METRIKA_ID` в ENV |
| Telegram Bot Token и Chat ID | `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` в ENV |
| Production домен | `NEXT_PUBLIC_SITE_URL` в ENV |
